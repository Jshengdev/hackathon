# Past Winners

**Real case studies of projects that won hackathons.**

## What You'll Find Here

Each case study includes:
- **What they built** — The core idea and implementation
- **Why it won** — What crossed The Gap
- **The wow moment** — The demo moment that made judges lean forward
- **Technical approach** — Stack, APIs, architecture
- **Track fit** — How they positioned for multiple tracks
- **Presentation** — How they pitched it
- **Team** — Size, skills, division of work
- **Timeboxing** — How they scoped to finish in time

## How to Use This Category

**When brainstorming:**
→ See what kinds of ideas actually win
→ Identify patterns across winners
→ Get inspired by moonshot thinking

**When validating your idea:**
→ Compare yours to winners
→ Check if you're crossing The Gap
→ See how similar ideas were positioned

**When preparing to demo:**
→ Study their presentation patterns
→ See how they framed the problem
→ Learn from their wow moments

## Adding Content

To add a past winner case study:

```bash
/bmad:hackathon-suite:librarian
```

Say: "Add a past winner case study"

The Librarian will ask for:
- Project name
- Hackathon name and year
- Prize/placement
- Track(s) it won
- Tags (AI, social-impact, hardware, etc.)
- Key insights
