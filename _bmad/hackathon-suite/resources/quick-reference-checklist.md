# Hackathon Quick Reference Checklist

**Print this or keep on your phone. Check off as you go.**

---

## ⏰ Timeline

- [ ] **Hour 0 (15 min):** Arrive, claim seat, settle in
- [ ] **Hour 0.25 (30-45 min):** Opening ceremony - TAKE NOTES
- [ ] **Hour 0.5-1.5 (60 min):** Sponsor table crawl - THE MOST IMPORTANT HOUR
- [ ] **Hour 1.5-2 (30 min):** Team yap mode / brain dump
- [ ] **Hour 2 (15 min):** Run `/bmad:hackathon-suite:context-collector`
- [ ] **Hour 2-3:** Ideation + validation workflows
- [ ] **Hour 3+:** BUILD

---

## 📝 Opening Ceremony Capture

For **EACH track**, note:
- [ ] Track name (exact wording)
- [ ] Sponsor company
- [ ] Prize amount ($$$)
- [ ] Official criteria (what they SAY)
- [ ] What they got excited about (body language)
- [ ] Examples they gave
- [ ] What they repeated/emphasized

Also capture:
- [ ] Submission deadline (EXACT time)
- [ ] Demo format (in-person/video/both)
- [ ] Devpost URL
- [ ] Judge names + backgrounds

---

## 🎯 The 6 Questions (At Every Sponsor Table)

Ask these at EVERY table:

1. **"What does a WINNING project look like for this track?"**
   - [ ] Captured their answer

2. **"Between [Criteria A] and [Criteria B], which matters more?"**
   - [ ] Captured their priority

3. **"What would make you lean forward and go 'whoa'?"**
   - [ ] Captured The Gap signal

4. **"What mistakes have you seen teams make?"**
   - [ ] Captured antipatterns

5. **"Any APIs or tools you hope to see used?"**
   - [ ] Captured required tech

6. **"Can I grab your contact for questions later?"**
   - [ ] Got contact info / business card

---

## 📋 After Each Table (Immediate)

Write down NOW (before you forget):
- [ ] Company name + track name
- [ ] Contact name + email
- [ ] What they ACTUALLY want (2-3 bullets)
- [ ] The Gap quote ("We'd be excited if...")
- [ ] Required tech/APIs
- [ ] Antipatterns to avoid
- [ ] Your vibe check (1-5 stars)

---

## 👥 Team Formation Checklist

If finding teammates:
- [ ] What tracks are you interested in?
- [ ] What's your background? (dev/design/business)
- [ ] What's your energy level?
- [ ] What have you built before?
- [ ] How do you handle pressure/deadlines?

Red flags to avoid:
- [ ] Know-it-alls who won't listen
- [ ] Wants to "lead" but won't build
- [ ] Zero energy/enthusiasm
- [ ] Skill overlap (3 backend devs, no frontend)

---

## 💬 Yap Mode Capture

During team brain dump, capture:
- [ ] Every "what if..." idea (no filtering!)
- [ ] Every "wouldn't it be cool..." thought
- [ ] Reactions to different tracks
- [ ] Skills/tools people are excited to use
- [ ] Hardware/APIs available
- [ ] Time constraints acknowledged

**Pro tip:** Voice record this session

---

## ✅ Pre-Ideation Verification

Before running idea-generator, confirm:

**Must-Have:**
- [ ] All major track criteria documented
- [ ] 3+ sponsor conversations captured
- [ ] Team skills/constraints noted
- [ ] Submission deadline confirmed
- [ ] Demo format understood

**Should-Have:**
- [ ] "What excites you" insights from sponsors
- [ ] Past winner examples
- [ ] Judge backgrounds
- [ ] Track-stacking opportunities identified
- [ ] Yap mode notes

**Ready? Run:** `/bmad:hackathon-suite:context-collector`

---

## 🚨 Common Mistakes to Avoid

- [ ] ❌ Starting to code before context gathering
- [ ] ❌ Skipping sponsor table conversations
- [ ] ❌ Just grabbing swag without talking
- [ ] ❌ Not taking notes (relying on memory)
- [ ] ❌ Missing opening ceremony
- [ ] ❌ Ignoring submission deadline
- [ ] ❌ Building for no specific track
- [ ] ❌ Overscoping ("Let's build everything!")

## 🏆 Networking Quick Wins

During hackathon:
- [ ] Get 3+ business cards from sponsors
- [ ] Connect with teammates on LinkedIn
- [ ] Thank organizers when you see them
- [ ] Visit sponsor tables during the hack (show progress)
- [ ] Connect with interesting neighboring teams

After hackathon:
- [ ] Follow up with sponsors within 48 hours
- [ ] Thank mentors who helped
- [ ] Add teammates to network
- [ ] Join hackathon alumni Discord/Slack

---

## ⚡ Emergency 15-Min Context Gathering

If you're late or time-crunched:

1. **Devpost (5 min):**
   - [ ] Read all track descriptions
   - [ ] Note prize amounts
   - [ ] Check submission deadline

2. **Top 3 Tables (7 min):**
   - [ ] Visit your top 3 target tracks
   - [ ] Ask only: "What would make you go 'whoa'?"
   - [ ] Get contact info

3. **Quick Yap (3 min):**
   - [ ] Team agrees on 1-2 track targets
   - [ ] Quick skills assessment

**Then run context-collector ASAP**

---

## 📊 Information Priority

**Capture in this order if time is limited:**

1. **Track criteria** (what judges want)
2. **Sponsor hidden priorities** (The Gap insights)
3. **Submission deadline** (hard constraint)
4. **Demo format** (shapes your build)
5. **Team skills** (feasibility)
6. **Required tech** (APIs/tools)
7. **Past winners** (pattern recognition)
8. **Judge backgrounds** (pitch tailoring)

---

## 🎯 The One Rule

**"Don't write a single line of code until you've run context-collector and idea-generator."**

Context first. Code second. Win.
